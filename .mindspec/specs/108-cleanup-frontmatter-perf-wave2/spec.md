# Spec 108-cleanup-frontmatter-perf-wave2: Cleanup wave 2: frontmatter unification, validation perf caching, deferred trace/golangci ownership

## Goal

Land the second bounded cleanup wave from the 2026-07-02 repo review (`review-source-report.md`, §4 order 4–5 plus the two ownership-gated items wave 1 explicitly deferred), scoped to exactly four work areas: (1) claim the two unowned paths wave 1 could not touch — `internal/trace/**` and `.golangci.yml` — in the workflow domain's `OWNERSHIP.yaml`, then delete the dead `trace.Event.MarshalJSON` and drop the three stale `unparam` carve-outs those claims unblock (the `findRoot` one relies on spec 107 landing in this branch's base first — see R11); (2) collapse the re-implemented YAML-frontmatter fence scanners onto the canonical `internal/frontmatter` package (report §2 DRY #3), including replacing the `## Approval` prose-scan with the frontmatter-based `validate.SpecStatus` so a spec has one source of approval truth; (3) hoist per-`(file × domain)` `OWNERSHIP.yaml` loading and memoize per-run ADR parsing in `internal/validate` (report §3 Perf #1 + #4); (4) make `doctor`'s dead-manifest check walk the repo tree once instead of once per domain (report §3 Perf #5). Target outcome: one source of truth for frontmatter and approval status (a documented ZFC correctness fix), materially fewer file reads and git subprocesses on the validation and doctor hot paths, and two previously-unowned paths brought under a domain — with byte-identical externally observable behavior everywhere except the deliberately-tightened frontmatter fence strictness (resolved to the canonical package's semantics) and the corrected approval-status source of truth.

## Background

The 2026-07-02 review swept 341 tracked Go files with `deadcode` (with and without `-test`), `golangci-lint`, and four subsystem review agents, verifying every finding against source (the panel-local tracked copy is `review-source-report.md`). It ranked a cleanup order (§4). Wave 1 (spec 107, `review/spec-107-approve/source-report.md` §4 order 1–3) landed the dead-code sweep, setup managed-block unification, and the `complete` perf pair, and **explicitly deferred to wave 2** the frontmatter consolidation, the ownership/ADR validation caching, the doctor per-domain walk, the `internal/trace` dead-symbol deletion, and the stale `.golangci.yml` `unparam` carve-out removals. It deferred the trace and `.golangci.yml` edits because `internal/trace` is claimed by zero manifests and `.golangci.yml` is a non-doc file matching no `OWNERSHIP.yaml` glob (and, unlike `AGENTS.md`, it is not in `isDocFile`'s `rootOperatorDocs` allowlist), so editing either path while unowned would trip `adr-divergence-unowned` at `mindspec complete` (wave-1 spec 107 § Out of Scope). This spec is **wave 2**: it lands those deferrals by claiming the paths *first*, then makes the deferred frontmatter and perf changes.

- **Frontmatter re-implementation (§2 DRY #3)** is the highest-correctness item: `internal/frontmatter.Parse` is documented as the single implementation ("callers must NOT re-implement… they will silently drift"), yet `internal/approve/plan.go` carries two ~40-line read-mutate-rewrite scanners (`updatePlanApproval`, `writeBeadIDsToFrontmatter`) and four more sites hand-roll the `---` fence scan (`approve/spec.go` `splitFrontmatter`, `approve/impl.go` `readPlanBeadIDs`, `contextpack/budgeter.go` `parseCitedADRs` + the `extractPlanBeadSection` strip, `validate/plan.go` `parsePlanFrontmatter`). The drift is real and observable: the canonical `Parse` requires `strings.TrimRight(line, "\r\n") == "---"` while the hand-rolled copies accept a `strings.TrimSpace`d fence, so a space-padded fence line parses differently depending on which copy runs. Separately, `validate/state.go` `readSpecApprovalStatus` substring-scans the `## Approval` prose that the YAML-frontmatter `validate.SpecStatus` was created to replace — two sources of truth that can disagree on the same `spec.md` (the ZFC violation the `frontmatter` package documents against).
- **Validation perf caching (§3 Perf #1 + #4)** targets the divergence hot path: `attributeDomain` calls `loadOwnershipForRef` once per domain, invoked once per changed file, so one gate run does `N × D` manifest parses — each a `ReadFile` + `yaml.Unmarshal`, or up to three `git show` subprocesses per domain when reading at a ref (`LoadOwnershipAtRef`). `checkUnclaimedSource` (`docsync.go:220-230`) already shows the fix: load every manifest once, attribute in memory. In the same file family, plan validation re-parses cited ADRs `O(domains × citations)` because `coverageOf`/`hasAcceptedCitation`/`checkADRCitations` each call `store.Get` per citation per domain and every `Get` re-reads and re-parses the ADR file.
- **Doctor per-domain walk (§3 Perf #5)** is a self-contained fix: `manifestResolvesAny` (`doctor/ownership.go:121-153`) does a full `filepath.WalkDir(root)` per domain, and the dead-manifest case it exists to catch always walks to completion → `O(domains × repo files)`.

Everything else the review surfaced (the remaining DRY items #2/#4/#6–#18 and Perf #2/#3/#6/#7) is deliberately **out of scope** (see Out of Scope) so this wave stays bounded and independently reviewable against the report.

## Impacted Domains

- **workflow** — owns all four work areas' edits: `internal/approve/**` (frontmatter mutate/read consolidation), `internal/validate/**` (frontmatter reader migration, `readSpecApprovalStatus` replacement, `OWNERSHIP.yaml` load hoisting, ADR-parse memoization), `internal/doctor/**` (tree-walk-once), `internal/adr/**` (the memoizing `adr.Store` path, if the decorator wraps the store), `cmd/**` and `plugins/mindspec/**` (unchanged, but the domain root), plus the two paths this spec **newly claims** in `.mindspec/domains/workflow/OWNERSHIP.yaml`: `internal/trace/**` (the NDJSON tracer) and `.golangci.yml` (repo lint config).
- **context-system** — `internal/contextpack/**`: `budgeter.go`'s `parseCitedADRs` and the `extractPlanBeadSection` frontmatter-strip migrate onto `internal/frontmatter.Parse` (read-only migration; no change to deterministic budgeting output).

Attribution notes (ADR-0036):
- `internal/frontmatter/**` is a low-level shared package claimed by **no** `OWNERSHIP.yaml`. This spec consumes it read-only (imports `frontmatter.Parse`/`frontmatter.Field`); it is **NOT modified**, so it never enters a bead diff and never trips `adr-divergence-unowned`. All new frontmatter helpers live in owned packages (the mutate helper in `internal/approve`; readers call `frontmatter` directly).
- The trace-ownership decision is **workflow**, chosen honestly from the import graph rather than the report's prose: `internal/trace` is imported by `cmd/mindspec/root.go` + `cmd/mindspec/trace.go` (the `mindspec trace` subcommand and tracer init — `cmd/**` is entirely workflow-owned), `internal/instruct/run.go` (workflow), and `internal/bead/bdcli.go` (execution, a *consumer* that emits events). Two of the three importers and the entire owner-facing CLI surface are workflow; `execution`'s own overview explicitly disclaims CLI infrastructure and only owns git/process operations reached through the `Executor` interface, which trace is not. The review's "used by recording/otel" characterization is factually wrong (verified: `internal/recording` imports neither `internal/trace` nor `internal/otel`, and `internal/otel` does not import `internal/trace`), so nothing pulls trace toward core/observability. Workflow already owns the trace command; it should own the package behind it.
- `.golangci.yml` is repo lint tooling; workflow already owns the sibling repo-tooling paths (`scripts/bd-jsonl-merge-driver.sh`, `.claude/skills/**`, `cmd/**`, `plugins/mindspec/**`), so a one-line `paths:` claim there is the natural, minimal fix — chosen over extending `isDocFile`/the divergence classifier (which would be a broader semantic change to `internal/validate` source).

## Requirements

1. **Ownership claims land first.** Add `internal/trace/**` and `.golangci.yml` to the `paths:` list of `.mindspec/domains/workflow/OWNERSHIP.yaml`. This claim must be present **in the same diff as, or an earlier bead than,** any edit to source under `internal/trace/**` or to `.golangci.yml` — divergence reads each domain's `OWNERSHIP.yaml` at the diffed ref, so the claim only needs to be visible at that ref (same-diff-or-earlier is the exact invariant). Once it is, `validate.attributeDomain` returns `"workflow"` for `internal/trace/event.go` and for `.golangci.yml`, so neither trips `adr-divergence-unowned` when edited. Because `.mindspec/domains/workflow/OWNERSHIP.yaml` is itself workflow-owned non-doc source, the bead that edits it must also touch a workflow domain doc in the same diff (gate-forwardness; see R10).

2. **Delete `trace.Event.MarshalJSON`.** Remove the aliased no-op marshaler at `internal/trace/event.go:52-56`. It calls `json.Marshal(Alias(e))` where `Alias` is a defined type identical to `Event`, so it is byte-identical to Go's default struct marshaling and exists only as dead surface. After removal, `json.Marshal(trace.Event{...})` produces the same NDJSON bytes it does today for a representative populated and a zero-value event (proven by a golden test), and the tracer's emitted lines are unchanged.

3. **Remove all three stale `.golangci.yml` `unparam` carve-outs.** Delete the three exclusion entries: `internal/brownfield/plan.go` (`buildMigrationPlan` — package absent), `internal/contextpack/builder.go` (`isNeighbor` — function absent), and `internal/next/beads.go` (`findRoot`). Keep the live `internal/validate/state.go` (`validateReviewMode`) carve-out. The `brownfield` and `isNeighbor` entries already match nothing on the current tree and are inert today. The `findRoot` removal is safe **only because of the R11 prerequisite rebase**: once spec 107 (which deletes `findRoot`) is in 108's base, `findRoot` no longer exists in the tree, so its exclusion matches nothing and its removal is inert too — the implementing bead MUST verify `internal/next/beads.go` contains no `findRoot` before deleting that carve-out. This ordering matters because `unparam` DOES flag `findRoot` while it is still present: removing the `findRoot` exclusion **while `findRoot` remains in the tree** makes `golangci-lint run` report `internal/next/beads.go:53: findRoot - result 1 (error) is always nil` (probed via `golangci-lint run --no-config --enable=unparam ./internal/next/...`, where `--no-config` bypasses ALL exclusions rather than removing this one). After the removals with 107 in base, `golangci-lint run ./...` is clean.

4. **Consolidate the two approve mutate-rewrite scanners.** Collapse `internal/approve/plan.go`'s `updatePlanApproval` (:207-268) and `writeBeadIDsToFrontmatter` (:686-745) — near-identical read → find-`---`-fences → parse-block-to-map → mutate → re-marshal → splice → write flows differing only in the mutation — into one helper in `internal/approve` built on `internal/frontmatter.Parse` (locate the block via `Parse`, `yaml.Unmarshal` it into `map[string]interface{}`, apply a caller-supplied mutation, `yaml.Marshal`, and splice the re-marshaled block ahead of the exact post-fence body bytes). `internal/frontmatter` is NOT modified. For well-formed inputs the helper writes byte-identical files to today's two functions (proven by golden tests for both the approval-fields write and the `bead_ids` write).

5. **Migrate the reader fence-scans onto the canonical package.** Replace the hand-rolled `---` fence scans in `internal/approve/spec.go` (`splitFrontmatter`, :270-288), `internal/approve/impl.go` (`readPlanBeadIDs`, :440-466), `internal/contextpack/budgeter.go` (`parseCitedADRs` :581-608 and the `extractPlanBeadSection` frontmatter-strip :614-631), and `internal/validate/plan.go` (`parsePlanFrontmatter`, :266-303) with calls to `internal/frontmatter.Parse`/`frontmatter.Field`. The known fence-strictness difference (the copies' `strings.TrimSpace` vs the canonical `strings.TrimRight(line, "\r\n")`) is resolved **to the canonical `Parse` semantics** — a deliberate tightening: a space-padded fence line (e.g. `  ---  `) is now treated as no-frontmatter everywhere, matching `frontmatter.Parse`. Redundant manual `#`-comment-line filtering is dropped (YAML's parser already ignores comments). A test demonstrates the tightening (a space-padded fence now yields no-frontmatter through the migrated path). For the `internal/contextpack` migration specifically, correctness is anchored by the existing deterministic-output coverage: `BuildBead("bead-fixture", …)` output must stay byte-identical, verified by `TestContextPackDeterministic` (`internal/contextpack/budgeter_test.go`) and `TestContextPackFlatVsCanonicalByteIdentical` (`internal/contextpack/budgeter_layout_test.go`), which transitively exercise `parseCitedADRs` and the `extractPlanBeadSection` strip — both must continue to pass unchanged.

6. **One approval-status source of truth.** Delete `internal/validate/state.go`'s `readSpecApprovalStatus` (:189-215) prose-scan of the `## Approval` `**Status**:` line and route its two callers (`state.go:70` and `state.go:111`, currently comparing `== "APPROVED"` / `!= "APPROVED"`) through the frontmatter-based `validate.SpecStatus`, comparing case-insensitively to `"Approved"`. Update the existing test at `internal/validate/state_test.go:200`. After the change, the declared spec status comes only from the YAML frontmatter `status:` field; a test proves that when the frontmatter `status:` and the `## Approval` prose disagree, the frontmatter value decides (the ZFC correctness fix).

7. **Hoist OWNERSHIP.yaml loading in validate (Perf #1).** In `internal/validate`, load each candidate domain's `Ownership` once per `ValidateDivergence` (`divergence.go`), `checkInternalPackages` (`docsync.go:459`), and `normalizeImpactedDomains` (`ownership_resolve.go:72`) call into an in-memory map keyed by domain, and attribute changed files against that map instead of calling `loadOwnershipForRef` per `(file × domain)` — the pattern `checkUnclaimedSource` (`docsync.go:220-230`) already uses. This eliminates the per-file re-parse and the up-to-three `git show` subprocesses per domain in `LoadOwnershipAtRef`. Attribution results (owning domain, `Ownership`, and every emitted error/finding) are identical to today — proven by the shared golden-diagnostics fixture test (see Acceptance Criteria, covering R7 and R8). Route per-domain manifest loading through a countable seam (a package-level load counter or an injectable loader func) so a test can assert that, for a multi-file diff, **each of the three call sites — `ValidateDivergence`, `checkInternalPackages`, and `normalizeImpactedDomains` — loads each domain's manifest a number of times that is a function of domain count only, independent of the number of changed files** (three seam-counting tests, one per site).

8. **Memoize ADR parsing per validation run (Perf #4).** Ensure the cited ADR set is parsed at most once per validation run — either by pre-parsing the cited IDs into a map or by wrapping `adr.Store` in a memoizing decorator (living in `internal/validate` so `internal/adr` need not change) — so `coverageOf`, `hasAcceptedCitation`, and `checkADRCitations` stop re-reading each ADR file `O(domains × citations)` times. Every coverage, relevance (`adr-cite-irrelevant`), and supersede-chain decision, and every emitted error/warning, is byte-identical to today — proven by the shared golden-diagnostics fixture test (see Acceptance Criteria, covering R7 and R8). Each distinct cited ADR is read from disk at most once per run (proven by a counting-`Store` test).

9. **Doctor walks the tree once (Perf #5).** Refactor `internal/doctor/ownership.go` so the dead-manifest check enumerates the live workspace file list once per ownership check and tests each domain's `paths:` globs against that cached list, replacing the per-domain `manifestResolvesAny` full walk. Route the single enumeration through a **package-level seam `walkWorkspaceFn = filepath.WalkDir`** (the current call is `filepath.WalkDir(root, …)` at `ownership.go:128`, honoring the existing `walkExclusions` for `.git/`, `.worktrees/`, `.beads/`) so a test can count its invocations. Warn/pass results are identical to today: a fixture-repo test asserts the full doctor `Report` set (per-domain check code + status) is unchanged across populated, empty-stub, and dead-manifest fixture domains. The tree is walked exactly once per check regardless of domain count, asserted by counting `walkWorkspaceFn` invocations.

10. **No regressions; gate-forwardness inherited by planning.** `go test ./...` passes with all existing tests green plus the new tests named in the Acceptance Criteria; `mindspec validate spec 108-cleanup-frontmatter-perf-wave2` reports only the `lifecycle-binding` warning. Planning MUST honor two ordering/attribution constraints baked in here: (a) work area 1's `OWNERSHIP.yaml` claims are present **in the same diff as, or an earlier bead than,** any edit to `internal/trace/**` or `.golangci.yml` — divergence reads each domain's `OWNERSHIP.yaml` at the diffed ref, so the claim only needs to be visible at that ref (same-diff-or-earlier is the exact invariant) — and therefore no bead diff ever contains an unclaimed non-doc path (`adr-divergence-unowned`); and (b) every bead's diff touches a domain doc for **each** domain whose non-doc source it changes (a workflow doc when it changes workflow source, a context-system doc when it changes `internal/contextpack`), because divergence attributes every changed non-doc file and requires the owning domain's docs to move with it.

## Non-Goals

- This spec does not redesign the frontmatter, ownership, ADR, or doctor subsystems; it removes duplication and redundant I/O within their existing contracts.
