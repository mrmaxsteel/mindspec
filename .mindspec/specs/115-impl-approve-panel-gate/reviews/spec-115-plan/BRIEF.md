## What the plan does

Spec 115 closes the last un-gated bead-branch merge path in the binary: `mindspec impl approve` gains a pre-terminal REFUSAL gate so it cannot finalize a spec while any closed bead under the spec's epic lacks proof of panel settlement. The design survived a 7-round spec marathon; the load-bearing resolutions the plan must honor:

- **C+B (no exit-code classifier).** Git cannot distinguish "ref absent" from "ref-store unreadable" by a single ref-probe's exit code (proven 9/9; see bd memory `git-ref-probe-exit-code-impossible`). So detection uses the UNCHANGED bool `gitutil.BranchExists` (false = absent OR probe-infra-failure = no trigger, by design), and fail-closed safety is STRUCTURAL: (Fact 1) `exec.MergeBase("main", specBranch)` at `impl.go:249` aborts the whole pre-scan on an unreadable store; (Fact 2) `FinalizeEpic` merges only worktree-enumerated real branches. There must be NO `BranchExistsE` and NO `show-ref` Go code anywhere (negative fence, 0-hit at `8020b3e2`), and `internal/gitutil` must be byte-unchanged.
- **Option B (round-7, the TOCTOU fix).** A transient `BranchExists` miss could desync detection (probe) from merge (`WorktreeOps.List()`). Fix: the gate ALSO enumerates via `bead.WorktreeList()` — the SAME source `FinalizeEpic` merges from — and refuses on any enumerated closed-epic-bead non-ancestor `bead/<id>` worktree, ADDED to (not replacing) the `BranchExists`/shared-predicate detection. Disclosed residual: the gate's `List()` and `FinalizeEpic`'s `List()` are two temporally-separated calls; a worktree created between them is concurrent-lifecycle-mutation, scoped out under mindspec's serialized one-mutator model.
- **Single settlement home.** Recovery routes through the ONE existing gate, `mindspec complete <bead>`; `impl approve` REFUSES but never computes an Allow/Block decision, never writes panel metadata, never applies a refutation (ADR-0037 reach extension, not a second gate home). ADR-0030-clean: no executor seam, no import cycle.
