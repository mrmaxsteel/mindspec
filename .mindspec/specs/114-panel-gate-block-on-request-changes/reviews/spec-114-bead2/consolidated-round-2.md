## 1. [BLOCKING — G1 + F1, both E2E-reproduced] Origin-unbound discharge → false "resolved naturally" audit + merge
**The hole**: a `refutation_pending` entry stores only `slot+round`, NOT which panel it originated from. `reconcilePendingRefutations` re-scans only *currently-surviving* matched panels and discharges when every one resolves the slot. So an operator can settle a genuine refutation obligation as a no-questions-asked "naturally moot" discharge (which needs no reason/evidence) by making the origin panel disappear while a decoy resolved panel remains:
- **Removal (G1)**: run 1 — `aaa-first/X=APPROVE` + `zzz-origin/X=REQUEST_CHANGES`, refutation X@1 applied, marker `refutation_pending X@1` durably recorded, close/audit fails → bead `in_progress`. Run 2 — DELETE `zzz-origin/panel.json`, keep `aaa-first/X=APPROVE`, under `MINDSPEC_SKIP_PANEL`. Both gate and reconcile now scan the singleton `{aaa-first}`; discharge fires, `refutation_discharged X@1` written, bead closes + merges. The origin RC never resolved.
- **Corruption (F1)**: same, but instead of deleting `zzz-origin/panel.json`, CORRUPT it — `panel.ForBead` drops registrations with `Err != nil`, so a malformed origin panel silently leaves the matched set exactly like removal. This is the precise hand-edit the refutation feature invites.
- **Between-scans (G1)**: origin disappears between the gate scan and the reconcile scan → same loss.
Removing ALL panels Refuses ("restore the panel"), but removing SOME discharges — an inconsistent, exploitable defense. This weakens the AC11 invariant: a durably-cleared RC settles without a `panel_refuted` (reason+evidence) audit and without a *verified* natural resolution.

**Fix (bind each pending obligation to its origin panel identity/source set)**:
1. **At persist time** (`persistRefutationPending`, in `panelGate`): each `refutation_pending` entry must record the ORIGIN panel identity — the set of panel slugs/dirs where that `(slot, round)` refutation actually cleared a latest-round REQUEST_CHANGES (i.e. the panels whose tally had `slot`=RC at `round` that the applied refutation cleared). Build this where `panelGate` aggregates `AppliedRefutations` across matched registrations — it knows which registration each applied entry came from. Add an origin field to the pending entry schema (e.g. `panels: ["<slug>", ...]`). One entry may have several origins if the same slot+round RC was cleared in multiple matched panels; record all.
2. **At reconcile**: for a pending `(slot, round)` with recorded origin set O, DISCHARGE only if EVERY panel in O is (a) present in the current matched set, (b) readable (`Registration.Err == nil`), AND (c) affirmatively resolves the slot per the two-disjunct test (`LatestRound > round`, or `==round` & the slot's latest verdict present-and-not-RC). If ANY origin in O is absent, unreadable, or still shows the slot as a live RC → do NOT discharge → Satisfy-if-applied-this-run, else REFUSE (recoverable: "restore the origin panel(s) so the obligation can be discharged or satisfied"). A decoy non-origin panel can NEVER substitute for a missing/corrupt origin.
3. **Legitimate natural resolution still works** (AC4b, zero-ceremony re-panel): the origin panel persists and its re-tally advances (`LatestRound > round`) or the slot flips to APPROVE at the same round → discharge fires against the actual origin. Verify this still passes.
4. **Fail-closed origin decode**: a pending entry missing its origin set, or with a malformed origin value, must REFUSE (can't verify against origin) — symmetric with fix #3.

**Tests (required, name under `TestPanelRefuted_*`)**:
- `TestPanelRefuted_OriginRemoved_Refuses` — origin panel deleted before retry while a decoy resolved panel remains → Refuse, NO discharge audit, NO close/merge. Positive control: origin present + resolved → discharge.
- `TestPanelRefuted_OriginCorrupted_Refuses` — origin `panel.json` corrupted (parse error / `Err != nil`) with a decoy present → Refuse.
- Mutation-proof: reverting the origin-binding (discharge against currently-matched instead of recorded origins) must turn `TestPanelRefuted_OriginRemoved_Refuses` RED.
