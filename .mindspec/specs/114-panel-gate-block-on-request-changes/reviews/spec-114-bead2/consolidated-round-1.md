## 1. [BLOCKING — G1] Cross-panel false discharge
**Evidence (G1, reproduced in /tmp)**: `internal/complete/panel_advisory.go` — the gate selects the FIRST matched registration (~:178-186) but aggregates applied refutations across ALL matched registrations (~:231-238); `reconcilePendingRefutations` (~:584-604) re-tallies ONLY that first directory. Repro: two panel dirs match one bead — `aaa-first/` where slot X is APPROVE, `zzz-origin/` where slot X is REQUEST_CHANGES — under `MINDSPEC_SKIP_PANEL`, reconciliation discharges the pending X@1 from `aaa-first/`'s APPROVE state while X's RC is still live in `zzz-origin/`. An audit obligation is discharged without the RC actually being resolved → violates the AC11 verified-discharge invariant.

**Fix**: a pending `(slot, round)` may be DISCHARGED only when it is affirmatively resolved in the panel that could carry it — never from an arbitrary first-matched panel. Two acceptable designs (pick the cleaner in-code):
- **(preferred, simplest-safe)** Reconciliation re-tallies EVERY matched panel registration for the bead (not just the first). Discharge a pending `(slot, round)` only if NO matched panel shows `slot` as a latest-round `REQUEST_CHANGES` at/covering `round` (i.e. the two-disjunct discharge test — LatestRound>round, or ==round & slot's latest verdict present-and-not-RC — must hold in EVERY matched panel that contains that slot; if ANY matched panel still shows it live, do NOT discharge → Satisfy-if-applied-this-run, else Refuse). This makes discharge conservative and correct regardless of which panel the obligation came from. panelGate must therefore surface ALL matched registrations (or their dirs) to reconciliation, not just the first — extend its return (or have reconcile re-scan `<spec-dir>/reviews/*/panel.json` for the bead_id) accordingly.
- **(alt)** Bind each `refutation_pending` entry to its originating panel dir at marker-write time (record the dir/slug in the entry) and verify discharge against THAT panel only.
**Test (required)**: a multi-panel hatch/no-panel retry e2e with contradictory same-slot verdicts across two panel dirs — assert the pending is NOT discharged while its origin RC is live (Refuse), and IS discharged only once the origin panel's slot resolves. Name it under the `TestPanelRefuted_*` prefix (e.g. `TestPanelRefuted_CrossPanel_NoFalseDischarge`).

## 2. [MAJOR — G1, O1, F1] Already-covered pending not recognized (plan L534 deviation)
**Evidence**: `reconcilePendingRefutations` (~:566-628) never reads existing `panel_refuted_entries` / `refutation_discharged_entries` when reconciling. Plan L533-535 requires reconciling only entries "NOT already covered by a durable `panel_refuted`/`refutation_discharged` entry." Effect: a compound-failure retry — run 1 durably writes `panel_refuted(X,1)` but the close fails (2u0u non-durable close), then panel.json is removed before retry — wrongly REFUSES on retry even though the audit obligation is ALREADY durably met. Over-conservative (never loses an audit) but blocks a legitimate completion the plan intended to allow.
**Fix**: before the satisfy/discharge/refuse logic, subtract from the pending set every `(slot, round)` already covered by a matching durable `panel_refuted_entries` OR `refutation_discharged_entries` record on bead metadata (read via `completeGetMetadataFn`, fail-closed — a read error still Refuses per (d)). An already-covered pending reconciles to a no-op (its obligation is discharged-of-record).
**Test (required)**: a retry with a seeded covering `panel_refuted_entries` for X@1 and panel.json removed → reconciliation treats it as covered and COMPLETES (not Refuse). Name e.g. `TestPanelRefuted_AlreadyCoveredPending_Completes`.

## 3. [MINOR — F1, fold in] `decodePendingEntries` fail-open on malformed value
**Evidence (F1)**: `decodePendingEntries` treats a PRESENT-but-shape-corrupt `refutation_pending_entries` value (e.g. a string where an array is expected) as EMPTY — silently dropping any recorded obligation. That is the fail-OPEN direction (a corrupt obligation store lets the bead merge un-reconciled), contrary to this bead's fail-closed principle and symmetric with the (d) `GetMetadata` read-error ⟹ Refuse rule.
**Fix**: a present-but-malformed `refutation_pending_entries` (unmarshal/shape error) must surface an error that makes reconciliation REFUSE (fail-closed), NOT silently empty. A genuinely ABSENT key (no obligation) still reconciles to a no-op as today. (Same principle for a malformed `panel_refuted_entries`/`refutation_discharged_entries` when you read them for fix #2 — a present-but-corrupt value Refuses.)
**Test (required)**: a seeded malformed `refutation_pending_entries` → reconciliation Refuses (bead not closed). Name e.g. `TestPanelRefuted_MalformedPendingEntries_Refuses`.
