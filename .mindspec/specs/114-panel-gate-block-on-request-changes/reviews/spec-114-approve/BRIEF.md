## What this spec is
Enforces "a reviewer's REQUEST_CHANGES is never silently out-voted" IN THE BINARY. Today `mindspec complete`'s panel gate passes when APPROVE ≥ threshold (default N−1), no REJECT, no hard_block, SHA fresh — so one dissenting REQUEST_CHANGES is tolerated (out-voted by the count). Two operator-decided changes (already settled — assess soundness, don't re-litigate the decisions):
- **R1 (gate change)**: `mindspec complete`'s gate BLOCKS unless every expected reviewer's LATEST-round verdict is APPROVE or explicitly refuted; an unresolved REQUEST_CHANGES blocks exactly like a REJECT. **Layered model** (grill-resolved OQ1): the `approve_threshold`/N−1 floor stays a NECESSARY condition on GENUINE APPROVE count — a refuted RC clears the veto but NEVER counts as an APPROVE. Incomplete panels, stale SHA, hard_block, REJECT still block as today. Amends **ADR-0037**. Block message names the unresolved slot(s) + fix-and-re-panel `recovery:` line; does NOT advertise a paste-able refutation incantation (OQ4, mirrors MINDSPEC_SKIP_PANEL discretion).
- **R2 (audited-refutation escape)**: an operator clears ONE specific slot's RC by recording a refutation — a `refutations` array on `panel.json` (`{slot, round, reason, evidence}`, parse-lenient like `abandon_reason`) + a mandatory `panel_refuted` bead-metadata audit entry at complete (parallel to `panel_abandoned`). A refuted RC no longer blocks; the audit persists. Refutation ONLY clears a slot whose latest verdict is REQUEST_CHANGES — NEVER a REJECT/hard_block/incomplete/stale/sub-threshold; one refutation clears only its one named slot+round (a newer-round re-RC is not cleared). Hand-edit panel.json (abandon-parallel, agent-performable *because* audited); the GATE does the validation. A `mindspec panel refute` verb is a NON-GOAL/follow-up.
